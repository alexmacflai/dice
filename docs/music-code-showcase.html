Scales, Weighted Scales, and Octaves

This page collects the music-generation code used in the project so the core musical logic can be shown clearly in documentation or captured as a clean portfolio image. The excerpts below come directly from the music engine.

Source: src/audio.ts

Scales

The engine defines three pitch collections as semitone offsets from a base MIDI note. Each mode points at one of these scales through its profile. 

const MAJOR = [0, 2, 4, 5, 7, 9, 11];
const NATURAL_MINOR = [0, 2, 3, 5, 7, 8, 10];
const PHRYGIAN = [0, 1, 3, 5, 7, 8, 10];

const MODE_PROFILES: Record<MusicMode, ModeProfile> = {
  soft: {
    bpm: 64,
    scaleSemitones: MAJOR,
    degreeWeights: SOFT_DEGREE_WEIGHTS,
    octaveWeights: MAJOR_OCTAVE_WEIGHTS,
    baseMidi: 56,
  },
  crisp: {
    bpm: 133,
    scaleSemitones: NATURAL_MINOR,
    degreeWeights: CRISP_DEGREE_WEIGHTS,
    octaveWeights: NATURAL_MINOR_OCTAVE_WEIGHTS,
    baseMidi: 82,
  },
  manic: {
    bpm: 101,
    scaleSemitones: PHRYGIAN,
    degreeWeights: MANIC_DEGREE_WEIGHTS,
    octaveWeights: PHRYGIAN_OCTAVE_WEIGHTS,
    baseMidi: 64,
  },
};
This is the scale layer: each mode chooses a pitch set before any weighting or octave drift is applied.

Weighted Scales

Scale degrees are not chosen uniformly. A weighted picker biases note selection so each music mode produces a distinct melodic personality. 

const SOFT_DEGREE_WEIGHTS = [0.30, 0.12, 0.20, 0.08, 0.18, 0.08, 0.04];
const CRISP_DEGREE_WEIGHTS = [0.28, 0.10, 0.22, 0.10, 0.18, 0.05, 0.07];
const MANIC_DEGREE_WEIGHTS = [0.32, 0.03, 0.23, 0.12, 0.18, 0.07, 0.05];

function weightedIndex(weights: number[]) {
  const total = weights.reduce((sum, weight) => sum + Math.max(0, weight), 0);
  if (total <= 0) return 0;

  let pick = Math.random() * total;
  for (let i = 0; i < weights.length; i++) {
    pick -= Math.max(0, weights[i]);
    if (pick <= 0) return i;
  }

  return weights.length - 1;
}

const degreeIndex = weightedIndex(profile.degreeWeights);
const semitone = profile.scaleSemitones[degreeIndex] ?? 0;
This is the weighted-scale layer: the engine picks a degree first, then resolves that degree inside the active scale.

Octaves

After the scale degree is chosen, the note is moved across weighted octave offsets. That keeps the harmony mode-specific while still giving the output vertical range. 

const OCTAVE_OFFSETS = [-24, -12, 0, 12, 24];
const MAJOR_OCTAVE_WEIGHTS = [0.15, 0.05, 0.55, 0.15, 0.05];
const NATURAL_MINOR_OCTAVE_WEIGHTS = [0.2, 0.1, 0.5, 0.3, 0.05];
const PHRYGIAN_OCTAVE_WEIGHTS = [0.16, 0.26, 0.36, 0.14, 0.08];

function weightedValue(values: number[], weights: number[]) {
  const idx = weightedIndex(weights);
  return values[Math.min(values.length - 1, Math.max(0, idx))] ?? values[0] ?? 0;
}

const octaveDrift = weightedValue(OCTAVE_OFFSETS, profile.octaveWeights);
const midi = profile.baseMidi + semitone + octaveDrift;
const freq = midiToHz(midi);
This is the octave layer: pitch height is selected after the scale degree, then converted from MIDI to frequency.
